Deployment Topology

Four independent projects with distinct deployment targets.

Deployment Options

Local Docker Compose

For development and testing

Run the entire stack locally with a single command. Infrastructure (Postgres+pgvector, Redis, NATS, MinIO) plus all OSS services.

  • cd knol-oss && docker compose up -d
  • cargo run --bin service-gateway
  • API at localhost:3000

Cloud Kubernetes

Production OSS deployment

Deploy the OSS services to any Kubernetes cluster. Use the provided Dockerfile for multi-stage builds. Managed databases recommended.

  • Dockerfile: multi-stage Rust builder
  • Horizontal scaling per service
  • Helm chart (coming soon)

Cloud Railway

Enterprise deployment

One-click deployment for enterprise services. Railway auto-detects the Dockerfile and Procfile for multi-process deployment.

  • railway login && railway up
  • 4 services: admin, jobs, billing, ingest
  • Auto-scaling and health checks

CDN Netlify

Marketing website

Static site deployment with Next.js static export. Connect your GitHub repo and Netlify builds automatically on push.

  • Build: npm run build → out/
  • Security headers via netlify.toml
  • Global CDN edge caching
