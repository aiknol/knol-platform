1. System Overview

Knol is an event-driven microservice platform that gives LLM applications persistent, structured long-term memory. Messages are ingested asynchronously, extracted into a knowledge graph by an LLM, stored with vector embeddings, and retrieved via adaptive hybrid search.

High-Level Architecture

Client / SDK → API Gateway :8080 → Write Service :8081 → NATS JetStream → Graph Service :8083 → PostgreSQL + pgvector
Client / SDK → API Gateway :8080 → Retrieve Service :8082 → PostgreSQL (Vector + BM25 + Graph)

Pattern: Write returns immediately (fast ACK). Entity/memory extraction happens asynchronously via NATS. Retrieval is synchronous, combining vector similarity, full-text BM25, and knowledge graph traversal using Reciprocal Rank Fusion.

The retrieve service classifies query intent using local heuristics (no LLM call needed) to weight the search strategies differently. For example, "Who does Alice work with?" triggers graph-heavy weights, while "What do I prefer for breakfast?" triggers vector-heavy weights.

7. LLM Optimizations (7 Layers)

1
Content Triage
Skip trivial messages (greetings, acks, short questions <3 words) before any LLM call. Classify content as Skip / Light / Full.
↓ 30-60% calls
2
Redis LLM Response Cache
Cache extraction results by SHA256(content + role + sorted_entities). Default TTL: 1 hour. Eliminates duplicate LLM calls entirely.
↓ 20-40% calls
3
Prompt Compression + Entity Pruning
Compact system prompt (~400 tokens). Prune entity context from 100 to only ~20 entities mentioned in content.
↓ 30-40% input tokens
4
Inline Verification
Embed grounded and ground_score fields in extraction prompt. Eliminates the separate verification LLM call.
↓ 50% calls (when verification enabled)
5
Dynamic Output Tokens
Scale max_output_tokens by content length: <50 words → 1024, 50-200 → 2048, 200+ → 4096. Prevents wasting output budget on short messages.
↓ 30-40% output tokens
6
Dynamic Provider Reload
DynamicLlmProvider re-reads API keys and provider config from DB every 60s. Change providers without restart.
Zero-downtime config
7
Token Usage Persistence
Every LLM call logs input_tokens, output_tokens, cache_hit to llm_usage_log table. Enables per-tenant cost monitoring.
Cost visibility

10. Authentication & Security

Tenant Auth (Public API)

Clients authenticate via X-API-Key header. The gateway hashes the key with SHA-256 and matches against tenants.api_key_hash. JWT Bearer tokens are also supported.

Admin Auth

Admin users log in with email/password (bcrypt). A JWT session is created with ~8h expiry, stored in admin_sessions table. Roles: super_admin, config_admin, marketing_admin, read_only.

Credential Encryption

LLM API keys and OAuth tokens are stored in system_credentials as AES-256-GCM encrypted bytes. The 32-byte key comes from ADMIN_ENCRYPTION_KEY env var (base64-encoded). A random 12-byte nonce is generated per encryption.

Security Features

  • Row-Level Security (RLS) — Every query scoped to current_setting('app.tenant_id')
  • PII Detection/Redaction — Regex-based detection of emails, phones, SSNs, credit cards. Configurable modes: redact, mask, hash, allow
  • Prompt Injection Detection — Heuristic detection before LLM calls
  • Rate Limiting — Per-tenant Redis-backed rate limiter in gateway
  • Full Audit Trail — memory_audit, admin_audit_log, marketing_audit, llm_usage_log

12. Enterprise Features

🔄 Memory Consolidation
Background job clusters similar episodic memories into semantic summaries. Configurable: min_age=24h, min_cluster=3, max_per_run=100.
⚡ Conflict Resolution
Detects contradictory memories and supersedes outdated ones. Duplicate threshold=0.80, contradiction threshold=0.50.
📉 Importance Decay
Exponential decay on importance scores over time. Lambda=0.01, min_importance=0.05. Keeps knowledge graph fresh.
📊 Usage Billing
Tracks write/search/extract/admin events per tenant. Enforces usage limits based on plan tier.
📣 Marketing Automation
Cron-scheduled publishing to Twitter, LinkedIn, Reddit, Dev.to, Email, GitHub. Redis-based rate limiting with 90% safety margins.
🔌 Ingestion Connectors
Webhook + bulk import framework for Slack, GitHub, email. Publishes to same NATS stream as write service.
