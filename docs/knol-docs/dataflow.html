Memory Types

EPISODIC

Episodic Memory

Raw conversation logs and interaction history. Stored chronologically with full context. The foundation for all other memory types — every interaction starts as an episode.

SEMANTIC

Semantic Memory

Consolidated facts, knowledge, and preferences. Created by the consolidation engine merging multiple episodic memories into durable, high-confidence facts. "User prefers dark mode."

PROCEDURAL

Procedural Memory

Learned workflows, patterns, and step-by-step procedures. Captures how users accomplish tasks. "When deploying, user always runs tests first, then builds, then pushes."

WORKING

Working Memory

Short-lived session context that persists within a conversation but expires after. Used for tracking current task context, active topics, and in-progress decisions.

Adaptive Retrieval Strategies

The Retrieve Service classifies each query's intent and selects the optimal combination of search signals.

Query Intent Example Primary Signal Secondary Signals RRF Weights
Preference "What does the user prefer?" Vector similarity Scope cascade Vector: 0.6, BM25: 0.1, Graph: 0.1, Scope: 0.2
Temporal "What happened last week?" Graph (temporal edges) BM25, Vector Vector: 0.2, BM25: 0.2, Graph: 0.5, Scope: 0.1
Relational "Who works with Alice?" Graph traversal Vector, Scope Vector: 0.15, BM25: 0.1, Graph: 0.6, Scope: 0.15
General "Tell me about the project" Hybrid (all) Equal weighting Vector: 0.3, BM25: 0.3, Graph: 0.2, Scope: 0.2

RRF Fusion Formula

score(m) = Σi wi / (k + ranki(m))
where k = 60, wi = intent-based weight for signal i, ranki(m) = position of memory m in signal i's results
