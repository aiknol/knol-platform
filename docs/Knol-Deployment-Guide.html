Knol deployment strategy

Cost-effective hosting for 8 Rust microservices, 4 infrastructure dependencies, and a static marketing site

Recommendation at a glance

Best overall: Hetzner VPS + Docker Compose for the backend (~€15/mo), Neon for Postgres (free→$5), Upstash for Redis (free→$2), Cloudflare Pages for the website (free). Total: ~$20–35/month to run the full Knol stack in production.

Why not Railway/Render? With 8 services, per-service pricing adds up fast. A single VPS running all containers is 3–5× cheaper at this stage.

Starter tier — detailed breakdown

This is the recommended starting point. You get a production-ready stack for under $25/month by combining a single Hetzner VPS with managed free tiers for Postgres and Redis.

Compute — Hetzner CX32

Hetzner CX32 — 4 shared vCPU, 8 GB RAM, 80 GB NVMe, 20 TB traffic€7.49/mo

All 8 Rust services + NATS + MinIO run on a single VPS via Docker Compose. Rust's tiny memory footprint makes this practical — total idle usage is ~700 MB including infrastructure containers, leaving 7+ GB for request handling.

Database — Neon serverless Postgres

Neon free tier — 100 CU-hours/mo, 0.5 GB storage, pgvector included$0

Neon is serverless Postgres with native pgvector support (critical for Knol's vector search). The free tier scales to zero when idle and provides 100 compute-unit hours per month — plenty for a pre-revenue product. When you outgrow it, Launch tier is pay-as-you-go at $0.106/CU-hour with no minimum.

Cache — Upstash Redis

Upstash free tier — 500K commands/mo, 256 MB$0

Upstash provides serverless Redis with a generous free tier. The HTTP-based API works well for cache workloads. At scale, the pay-per-request model ($0.20/100K commands) keeps costs proportional to actual usage.

Message queue — NATS on VPS

NATS 2.10 container on Hetzner VPS$0 (included)

NATS is lightweight (~30 MB RAM) and has no managed serverless offering worth using. Run it as a Docker container on the same VPS. JetStream provides persistence with minimal overhead.

Object storage — MinIO on VPS (or Cloudflare R2)

MinIO container on Hetzner VPS$0 (included)

For early stage, MinIO on the VPS works fine. When you need durability guarantees, swap to Cloudflare R2 (free egress, $0.015/GB storage).

Website — Cloudflare Pages

Cloudflare Pages — unlimited bandwidth, 500 builds/mo$0

The Next.js static export deploys perfectly to Cloudflare Pages. Unlike Netlify's 100 GB bandwidth cap, Cloudflare gives unlimited bandwidth on the free tier with a faster global CDN.

Starter total ~$8–22/mo

Range depends on Neon compute usage. Base cost is just the Hetzner VPS (~€7.49 ≈ $8). Add Anthropic API costs for Claude Haiku calls separately.

Platform comparison for compute

The key question is whether to use a VPS (self-managed containers) or a PaaS (managed deployment). Here's how the options compare for Knol's 8-service architecture:

Platform 8 services cost Setup effort Best for
Hetzner CX32 ~€7.49/mo Medium — Docker Compose + Caddy Cost-optimized production
Hetzner CAX21 (ARM) ~€5.49/mo Medium — needs ARM Docker builds Maximum savings
Railway ~$40–60/mo Low — git push deploy Fast iteration, solo dev
Fly.io ~$35–50/mo Low-Medium — Dockerfile deploy Edge deployment, global users
Render ~$50–70/mo Low — dashboard deploy Team-friendly dashboard
DigitalOcean Droplet ~$24/mo Medium — Docker Compose US-based, good docs
Why Hetzner wins for Knol: Rust microservices have tiny memory footprints (~20–30 MB each). PaaS platforms charge per-service, making them 5–8× more expensive for the same workload. A single 8 GB VPS easily runs all 8 services plus NATS and MinIO with room to spare.

Scaling migration path

Start cheap, scale incrementally. Here's the progression as Knol grows:

Phase 1 — Pre-revenue ($8–22/mo)

Single Hetzner CX32 with Docker Compose. Neon + Upstash free tiers. Cloudflare Pages. This runs the entire Knol stack including all enterprise services. Focus on shipping, not infrastructure.

Phase 2 — Early traction ($40–80/mo)

Upgrade to Hetzner CCX23 (dedicated vCPU) for consistent performance. Move to Neon Launch tier for more compute hours. Add Cloudflare R2 to replace MinIO for durable object storage. Add automated backups via Hetzner snapshots.

Phase 3 — Growth ($150–400/mo)

Split services across 2–3 VPS nodes. Move Postgres to a dedicated Hetzner server or Neon Scale tier. Add a second NATS node for HA. Consider k3s for orchestration. Add monitoring with Grafana + Prometheus (already in codebase).

Phase 4 — Scale ($500+/mo)

Kubernetes cluster on Hetzner (3 nodes). Managed Postgres with HA (Neon Business or Crunchy Bridge). Redis cluster on Upstash Pro. NATS cluster (3 nodes). CDN + DDoS protection via Cloudflare Pro.
